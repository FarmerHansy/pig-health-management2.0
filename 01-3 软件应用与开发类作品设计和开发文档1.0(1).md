# 第一章 需求分析

## 1.1 项目背景与问题

生猪养殖行业面临诸多挑战，包括疾病诊断依赖养殖经验、知识更新滞后、信息资源分散、用药过程缺乏可追溯性等问题。这些问题导致养殖效率低下、疾病防治不及时，并增加了经济损失风险。本项目旨在通过数字化手段，构建一个集疾病知识管理、药品信息维护、资讯学习和智能问诊于一体的智慧医药系统，实现生猪健康管理的智能化和规范化。

项目核心目标是将生猪疾病知识与人工智能大模型能力相结合，打造一个“可管理、可追溯、可扩展”的智能辅助诊断与知识服务平台。通过RAG（检索增强生成）技术，确保AI问诊基于可靠的领域知识，避免泛化回答，提升诊断准确性和实用性。

## 1.2 目标用户

- **普通养殖用户**：需要快速查询疾病信息、药品详情、相关文章，并通过AI问诊获取个性化建议。
- **兽医/技术人员**：利用系统辅助诊断、知识检索，并复盘历史问诊会话以积累经验。
- **管理员**：负责维护疾病库、药品库、文章资讯、知识库文件和用户数据，确保系统数据的准确性和完整性。

## 1.3 需求范围

1.  **账号体系与权限控制**：支持管理员和普通用户的注册、登录、权限分级管理。
2.  **基础业务管理**：包括疾病、药品、文章、资讯、用户反馈、文件上传等模块的增删改查。
3.  **AI问诊能力**：支持会话创建、多轮连续对话、历史记录追溯，确保问诊过程的连贯性和可追溯性。
4.  **RAG知识库能力**：支持多种格式文档上传、内容去重、自动入库和检索增强，提升AI回答的相关性和准确性。
5.  **多端访问支持**：Web管理端、Web用户端、小程序端共享同一后端服务，实现跨平台协同。

## 1.4 竞品与对标说明

本项目区别于通用问答产品，专注于生猪垂直领域的业务闭环：

- 与仅具备问答能力的通用大模型页面相比，本系统整合了疾病、药品、文章等业务数据，实现联动查询和推荐。
- 与传统信息管理系统相比，本系统引入RAG和多轮AI问诊能力，超越静态数据展示。
- 与单端产品相比，本系统提供管理端、用户端、小程序三端协同，覆盖从数据维护到用户应用的完整流程。
- 本作品对标****“畜安智诊”****（山东畜牧兽医职业学院）、“****数智兽医****”平台（百畜兴旺）及****LeeO系统****（默沙东动物保健）。竞品分析如下：

## 1.5 人工智能大模型应用说明

项目通过Spring AI同时支持本地Ollama与OpenAPI（默认配置为DeepSeek）两种调用方式。默认启用OpenAPI，按需切换为本地Ollama（qwen2.5:7b）并使用nomic-embed-text进行向量嵌入。问诊流程遵循RAG模式：先进行向量检索获取相关知识片段，再将检索结果与用户问题拼接生成提示词，最后调用模型生成回答；在仅启用OpenAPI时，知识库向量化将按配置关闭，以保证部署与成本可控。该方式确保回答基于领域知识、可追溯且更具可解释性。

# 第二章 概要设计

## 2.1 总体架构

系统采用前后端分离架构，确保模块化设计和可扩展性：

- **后端**：基于Spring Boot 3.3.5框架，集成MyBatis-Plus进行ORM操作，Sa-Token处理认证授权，Spring AI接入Ollama与OpenAPI双通道模型。
- **数据与中间件**：MySQL 8.x存储业务数据，Redis提供缓存和会话管理；通用文件采用MinIO对象存储，RAG知识库文件默认存放于resources/knowledge目录并入库索引。
- **前端**：
    - 管理端：Vue3 + Vite + Ant Design Vue + Element Plus，专注于运营维护界面。
    - 用户端：Vue3 + Vite + Vant + TailwindCSS，优化用户查询和问诊体验。
    - 小程序端：uni-app框架，支持跨平台移动应用部署。

架构设计遵循微服务思维，后端提供统一REST API，前端通过HTTP请求交互，确保数据一致性和安全性。

# 第三章 详细设计

### **3.1核心痛点分析**

****核心痛点****：

1.****兽医资源稀缺****：基层猪场均摊兽医不足，诊断依赖经验，误判率高

2.****症状描述非结构化****：农户多使用口语化表达（如“猪不吃食”“身上发红”），传统规则系统难以理解

****3.知识更新滞后****：疫病手册更新慢，新发变异毒株缺乏快速响应机制

4.****多源数据割裂****：影像、行为传感器、文本记录各自为政，无法形成诊断合力

****设计理念****：以RAG为核心引擎，采用多智能体协作架构，实现“口语入-证据出”的端到端智慧诊疗。

3.3数据库设计

数据库名称：pig_health_smart_medicine。  
包含15个核心数据表，设计遵循第三范式，确保数据完整性和查询效率：

- user（用户表）：存储用户基本信息、权限级别。
- article_types、articles（文章分类/文章表）：支持多级分类和富文本内容。
- news_articles（新闻资讯表）：动态资讯发布和管理。
- illness、illness_kind、illness_medicine（疾病体系表）：疾病分类、疾病详情、疾病-药品关联。
- medicine（药品表）：药品信息、规格、用法。
- conversation_session、conversation（会话与对话表）：支持会话级和消息级追溯。
- knowledge_file（RAG知识库文件表）：文件元数据、MD5去重。
- feedback、files、history、pageview（辅助表）：反馈收集、文件存储、操作历史、浏览统计。

关键设计要点：

1.  conversation_session与conversation分表设计，便于会话管理和消息分页。
2.  knowledge_file表使用file_md5字段实现内容去重，避免重复入库。
3.  通过illness_medicine关联表，实现疾病与药品的多对多关系，支持诊断推荐。

## 3.4关键技术实现

### 3.4.1 RAG知识初始化

系统启动时，KnowledgeInitializer组件在启用本地模型时扫描resources/knowledge目录，支持txt、md、pdf、docx格式文档。使用Apache PDFBox、POI等库解析文档内容，应用TokenTextSplitter进行文本分块，然后将向量嵌入写入SimpleVectorStore。该过程确保本地RAG知识库在应用启动后立即可用。

### 3.4.2 向量存储

采用SimpleVectorStore实现内存向量存储，适用于演示和轻量部署。配置与模型解耦，便于后续扩展到持久化存储如Pinecone或Milvus。嵌入模型使用nomic-embed-text，确保检索准确性。

### 3.4.3 AI问诊流程

1.  用户提交症状描述和可选会话ID。
2.  后端查询conversation_session获取上下文，在本地模型启用时执行向量相似检索。
3.  将检索到的知识片段与用户问题拼接为提示词，按配置调用Ollama或OpenAPI模型生成回答。
4.  将回答写入conversation表，更新会话状态，并可通过/provider接口查询当前生效模型信息。

该流程确保回答基于领域知识，提升可信度。

# 第五章 安装及使用

## 5.1 安装环境

- 后端：JDK 21、Maven 3.8+、MySQL 8.x、Redis
- 前端：Node.js 16+、npm
- 可选：Ollama服务、MinIO对象存储

# 第六章 项目总结

### **6.1开发感悟与团队协作**

本项目历时四个月，团队最大的感悟是“场景驱动胜过技术堆砌”。初期尝试直接调用通用大模型，但农户口语化描述（如“猪打蔫了”）与医学术语的鸿沟迫使我们必须引入RAG架构。任务分解上，我们将系统拆解为“症状采集→RAG检索→诊断推理→药品推荐→反馈闭环”五个环节，15张表的ER建模经过三轮评审才定稿——特别是conversation与conversation_session的分表设计，正是基于会话管理与消息分页的实际需求反复推敲得出。跨学科协作中，团队建立了“症状-疾病-药品”术语对照表，有效打通了兽医与计算机专业的沟通壁垒。

### **6.2克服的关键困难**

| 困难  | 解决方案 |
| --- | --- |
| 口语与术语匹配度低 | 构建生猪疾病领域同义词词典，查询时自动扩展 |
| LLM生成错误药品/剂量 | 实施“生成-检索”双重重排序，强制基于检索context作答 |
| 多模态特征融合困难 | 采用CLIP模型将图像特征映射到文本向量空间 |
| 知识文件重复上传 | knowledge_file表设置file_md5唯一索引自动去重 |

### **6.3升级演进规划**

****短期（1-3月）****：接入物联网设备数据（体温/采食量），适配微信小程序

****中期（3-6月）****：构建联邦学习框架实现跨猪场知识共享，增加疫病时空热力图预警

****长期（6-12月）****：探索“AI+专家”混合增强模式，开发药品库存联动与合规采购模块

### **6.4商业推广思考**

****目标用户****：存栏500-5000头的中型猪场技术员（痛点最强烈，付费意愿较高）。

****商业模式****：SaaS订阅制（基础版免费，专业版¥299/月）+ 专家会诊/药品集采等增值服务。****推广策略****：与省级农科院合作“科技下乡”试点，开发“猪病自诊”轻量版小程序免费引流。

### **6.5结语**

本项目的核心价值不在于技术前沿性，而在于证明了AI可以真正下沉到农业一线。当一位初中文化的养猪户通过手机拍照和语音描述就能获得接近执业兽医水平的诊疗建议时，技术才真正实现了普惠意义。未来我们将持续迭代，为生猪稳产保供贡献技术力量。
